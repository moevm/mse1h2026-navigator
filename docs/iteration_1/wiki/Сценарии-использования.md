
# Сценарии использования

## Часть 1. Авторизация и профиль

### UC-1.1: Вход через Яндекс ID
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: получить доступ к функционалу сервиса
- Система: идентифицировать пользователя

**Предусловие:** Пользователь не авторизован в системе

**Основной сценарий:**
1. Пользователь нажимает кнопку «Войти через Яндекс ID»
2. Система перенаправляет пользователя на страницу авторизации Яндекса
3. Пользователь вводит свои данные и подтверждает доступ
4. Система получает данные пользователя
5. Система создает сессию и перенаправляет пользователя на страницу поиска

**Результат:** Пользователь авторизован и может работать с сервисом

**Альтернативный сценарий**:
- 3a. Пользователь отказывается предоставить доступ или вводит неверные данные. Система возвращает пользователя на страницу входа с сообщением об ошибке

---

### UC-1.2: Просмотр и редактирование профиля
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: актуализировать список известных ему навыков

**Предусловие:** Пользователь авторизован

**Основной сценарий:**
1. Пользователь нажимает на иконку профиля на любой странице приложения
2. Система открывает страницу профиля
3. Пользователь нажимает «Добавить изученный навык»
4. Система открывает модальное окно с поиском навыков
5. Пользователь вводит название навыка и выбирает из предложенных вариантов
6. Система добавляет навык в список изученных

**Альтернативный сценарий:**
- 3a. Пользователь нажимает иконку удаления рядом с навыком. Система удаляет навык из списка

**Результат:** Список изученных навыков обновлен

---

## Часть 2. Поиск и построение графа

### UC-2.1: Поиск по целевой профессии
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: найти информацию о требуемых навыках для профессии

**Предусловие:** Пользователь авторизован и находится на странице поиска

**Основной сценарий:**
1. Пользователь вводит название целевой профессии в основное поле поиска (например, «ML Engineer»)
2. Пользователь при необходимости заполняет дополнительные фильтры (компания, навык, технология, источник курсов)
3. Пользователь нажимает кнопку «Построить граф»
4. Система выполняет поиск по указанным критериям
5. Система строит граф зависимостей навыков с учетом фильтров и изученных навыков из профиля
6. Система перенаправляет пользователя на страницу с графом

**Результат:** Пользователь видит интерактивный граф компетенций для целевой профессии

**Альтернативный сценарий:**
- 4a. По запросу ничего не найдено. Система отображает сообщение «Ничего не найдено»
- 4b. Указанный фильтр не найден. Система уведомляет об этом и строит граф без фильтрации

---

## Часть 3. Работа с графом компетенций

### UC-3.1: Просмотр и редактирование навыка
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: просмотреть детали навыка и настроить его параметры

**Предусловие:** Пользователь находится на странице с графом

**Основной сценарий:**
1. Пользователь кликает на узел графа
2. Система открывает модальное окно с информацией о навыке
3. Пользователь изменяет параметры (статус изученности, приоритет, обязательность)
4. Пользователь нажимает «Сохранить»
5. Система закрывает окно и обновляет отображение графа

**Результат:** Настройки навыка сохранены

**Альтернативный сценарий:**
- 3a. Пользователь нажимает «Удалить навык». Система удаляет узел из графа
- 4a. Пользователь нажимает «Отмена». Система закрывает окно без сохранения

---

### UC-3.2: Добавление нового элемента в граф
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: расширить граф дополнительными навыками

**Предусловие:** Пользователь находится на странице с графом

**Основной сценарий:**
1. Пользователь нажимает кнопку «Добавить элемент»
2. Пользователь вводит название навыка или выбирает из предложенного списка
3. Система добавляет новый узел в граф
4. Пользователь соединяет новый узел с существующими навыками, указывая зависимости
5. Пользователь заполняет параметры навыка и сохраняет

**Результат:** В граф добавлен новый навык с указанными связями

**Альтернативный сценарий:**
- 2a. Навык не найден. Система предлагает создать новый навык

---

### UC-3.3: Изменение порядка изучения
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: скорректировать последовательность освоения навыков

**Предусловие:** Пользователь находится на странице с графом

**Основной сценарий:**
1. Пользователь перетаскивает узел графа на новое место
2. Система проверяет сохранение логических зависимостей
3. Система фиксирует новое положение узла
4. Система пересчитывает общее время обучения

**Результат:** Порядок изучения навыков изменен

**Альтернативный сценарий:**
- 2a. Нарушены логические зависимости. Система возвращает узел на исходную позицию и уведомляет пользователя

---

## Часть 4. Экспорт результатов

### UC-4.1: Экспорт графа
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: сохранить граф во внешнем формате

**Предусловие:** Пользователь находится на странице с графом

**Основной сценарий:**
1. Пользователь нажимает кнопку «Экспорт»
2. Пользователь выбирает формат (RDF/OWL или изображение)
3. Система генерирует файл и инициирует скачивание

**Результат:** Пользователь скачал граф

---

### UC-4.2: Формирование и экспорт отчета
**Действующие лица:** Пользователь, Система

**Цели:**
- Пользователь: получить структурированный отчет по траектории обучения

**Предусловие:** Пользователь находится на странице с графом

**Основной сценарий:**
1. Пользователь нажимает кнопку «Сформировать отчет»
2. Система отображает предпросмотр отчета
3. Пользователь нажимает «Экспорт»
4. Пользователь выбирает формат (PDF или Markdown)
5. Система генерирует файл и инициирует скачивание

**Результат:** Пользователь скачал отчет
