## Описание проблемы

В современном IT-образовании существует разрыв между потребностями рынка и возможностями студентов по построению собственного обучения.

Основные затруднения студентов связаны с:
- Отсутствием видимости полной картины: cтудент понимает, какие предметы он изучает в семестре, но не видит, как они соотносятся с требованиями реальных вакансий
- Непониманием логики изучения: освоение навыка требует предварительных знаний, которые могут быть не описаны явно
- Информационной перегрузкой: cуществует множество образовательных платформ, курсов, книг и статей, но нет инструмента, который собрал бы данные ресурсы в единую структуру с учётом их взаимосвязей
- Несоответствием вакансий реальным требованиям: в описаниях вакансий часто указывается избыточный или несвязанный набор технологий, что затрудняет соискателям понимание действительно необходимых компетенций для работы

## Требования

### Функциональные требования

#### Поиск и навигация
- Пользователь может ввести целевую профессию произвольным текстом (не из фиксированного списка)
- Пользователь может выполнять поиск подграфа, фильтруя результаты по названию навыка, курса, технологии или компании
- Система строит подграф, релевантный запросу

#### Построение графа компетенций
- Граф отображает зависимости навыков, технологий и образовательных курсов, показывая, какие знания необходимы для каждой следующей ступени
- Минимальная единица графа (листовой узел) соответствует по объёму курсу на образовательной платформе или семестровому курсу вуза
- Для каждого навыка система предлагает список учебных материалов (онлайн-курсы, статьи, книги)
- При выборе компании подграф ограничивается только теми навыками и ресурсами, которые актуальны для конкретной организации
- Для каждого навыка и курса рассчитывается время обучения, которое суммируется в общий показатель

#### Взаимодействие с графом компетенций
- Граф отображается интерактивно с возможностью масштабирования
- Пользователь может отмечать уже известные навыки
- Пользователь может добавлять (выбор из предложенного списка или поиск по ключевому слову), заменять и удалять (скрывать) элементы графа
- Пользователь может изменять порядок изучения навыков, перетаскивая узлы графа
- Пользователь может задавать приоритеты навыков. Приоритеты навыков делятся на обязательные и опциональные. Пользователь может включать/выключать элементы по своему усмотрению

#### Сохранение данных, отчёты и экспорт
- Авторизация не обязательна; можно хранить данные локально (localStorage/IndexedDB) или экспортировать/импортировать файл с состоянием графа
- Пользователь может экспортировать построенный граф в стандартах RDF/OWL и в виде изображения
- Пользователь может получить отчёт в PDF/Markdown, который содержит: Целевая профессия; Ключевые навыки; Предварительные навыки/знания; Ресурсы для изучения; Оценки времени изучения

### Нефункциональные требования
- Для реализации алгоритмов на графах (топологическая сортировка для определения порядка изучения) рекомендуется использовать Python
- Использовать API hh.ru как источник информации по компаниям
- Время изучения брать из данных о курсах
- Для визуализации графа использовать React Flow
- Доступ к графу должен осуществляться через GraphQL
- Источники рекомендаций ресурсов: MOOCs (Coursera, edX, OpenEdu, Stepic  и др.), статьи, книги из Wikidata/DBpedia;

## Категории пользователей и их потребности

### Студенты

Заинтересованы в понимании того, каких навыков не хватает для трудоустройства. Хотят видеть порядок изучения и получать ссылки на конкретные курсы.

### Начинающие специалисты

Стремятся определить приоритеты для профессионального роста, систематизировать имеющиеся знания и понять, что изучать дальше.
