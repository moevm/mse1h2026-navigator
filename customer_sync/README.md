# Customer Sync Directory

## Созвон №1
- **Дата**: 19.02
- **Тема встречи**: Обсуждение требований к проекту
- **Обсужденные вопросы**:
  - **Выбор профессии из списка или пользователь вводит что захочет?**
  
    Пользователь вводит профессию вручную, а не выбирает из списка.
  - **Что имеется в виду под подграфом? Это уточнение изначального запроса пользователя деталями?**

    Поиск подграфа означает фильтрацию по навыку, курсу, технологии или компании. Должны отобразиться только узлы, связанные с искомым элементом.
 
  - **Каким образом пользователь должен осуществлять поиск подграфа? Ввод в поиске или через взаимодействие с узлом графа?**

    1.Через взаимодействие с узлом графа.

    2.Информация графа должна дублироваться в текстовом виде (список навыков), где пользователь сможет оставить только ту часть, которая ему интересна.

  - **API hh.ru подходит как источник информации по компаниям?**

    Да.

  - **Python это обязательный язык для бэкенда?**

    Можно использовать Python только для алгоритмов на графах. Для остального бэкенда можно использовать Express и TypeScript.
 
  - **Вид отчета?**

    Содержательную форму текстового отчета предлагается продумать самостоятельно, включив в него все собранные по запросу данные.

  - **Что подразумевается под экспортом результатов?**
 
    Экспорт текстового отчета в MD/PDF, экспорт изображения с графом, экспорт графа в RDF/OWL.

  - **Исходя из чего задавать приоритеты? Пользователь задает или они формируются в ходе построения графа?**

    Пользователь может включать и выключать элементы по своему усмотрению. Приоритеты ограничиваются категориями «опционально» или «обязательно». Возможность реализации зависит от доступных данных. Отдельно сказано о приоритетах как о фильтрации источников знаний (конкретный источник курса или книги).

  - **Нужно ли делать авторизацию? Можно ли использовать сторонние сервисы (например, Яндекс)?**
 
    Авторизация изначально не подразумевалась, но при необходимости сделать можно. Сторонние сервисы (например, Яндекс) использовать можно.

  - **Нужно ли углубляться в зависимости навыков?**
 
    Слишком глубоко не заходить, не ниже уровня курсов.

  - **Что попадает под понятие «предварительные навыки/знания»?**
 
    Предварительные навыки это требования к прохождению конкретных обучающих курсов, указанные на платформе с курсами. Если пользователь отметил навык как уже изученный, раскрывать его глубже не имеет смысла.

  - **Какой источник для времени изучения навыка?**
 
    Время берется из описания курсов. Время изучения навыка рассчитывается как сумма минимальных оценок времени по всем курсам, необходимым для его освоения.

  - **В чем заключается интерактивность с графом?**
 
    Отметка известных навыков, добавление и удаление элементов, выбор подграфа, установка приоритетов, изменение порядка перетаскиванием узлов графа.

  - **Что означает, что пользователь «может корректировать путь»?**
 
    Помимо описанного выше, пользователь может, например, заменить один курс на другой, после чего граф должен перестроиться соответствующим образом, а время изучения — пересчитаться.

    Оригинальный граф (до редактирования и до поиска подграфа) должен сохраняться, должна быть возможность вернуться к нему или получить отчет по нему.
